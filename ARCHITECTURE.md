# ארכיטקטורה

המטרה היא לא רק להעלים חלונות הרשאה, אלא לנתק את שרשרת העבודה מה-Mac המקומי ולהפוך את המערכת לאפליקציית מובייל/ווב עצמאית.

## אפשרות 1: אפליקציה עצמאית לחלוטין לטלפון

זו הארכיטקטורה שממומשת עכשיו בקוד:

- `PWA / Web App`:
  הממשק נטען בטלפון, ניתן להוסיף למסך הבית, ויש `manifest` + `service worker`.
- `Backend`:
  Flask API יוצר את קובץ ה-`DOCX` ואת ה-`PDF` בצד השרת.
- `PDF service`:
  ההמרה מתבצעת דרך `Gotenberg` או `LibreOffice headless`, לא דרך Word מקומי.
- `Source of truth`:
  קובץ ה-`DOCX` הוא המקור המחייב. ה-PDF מופק מתוך ה-DOCX, לא מתוך HTML/CSS.
- `Storage`:
  טיוטות ומסמכים נשמרים תחת `STORAGE_ROOT` בשרת או על volume קבוע.
- `פריסה מומלצת`:
  `app` + `gotenberg` + volume לאחסון קבצים + reverse proxy + HTTPS.

יתרונות:

- אין תלות ב-Mac אישי.
- הטלפון הוא סביבת העבודה הראשית.
- אפשר לפרוס על VPS, cloud VM או container platform.
- אין AppleScript, Automator או Word local automation.

חיסרון:

- ייתכנו פערי עימוד קטנים מול Word, כי ההמרה לא משתמשת במנוע Word.

## אפשרות 2: דיוק עימוד מקסימלי בלי חלונות הרשאה חוזרים

זו ארכיטקטורה נפרדת, לא המקומית על ה-Mac:

- הממשק נשאר אותו Web App / PWA בטלפון.
- ה-Backend נשאר צד-שרת.
- שירות ההמרה ל-PDF מתבצע על שרת מרוחק שמריץ Word או שירות conversion ייעודי עם fidelity גבוה מאוד.
- לחלופין, בסביבת בדיקה/שרת מקומי אפשר להשתמש ב-`Microsoft Word` שמותקן על השרת עצמו דרך `docx2pdf`, וזה נותן את אותו מנוע פריסה של Word.
- ה-DOCX נשלח לשירות `remote_word` ורק הוא יוצר את ה-PDF.
- הפונטים הקריטיים, כגון `David`, חייבים להיות מותקנים גם על שרת ה-Word המרוחק.
- כדי לזהות סטיות עימוד, ניתן להגדיר מנוע ייחוס (`PDF_COMPARE_REFERENCE_ENGINE=remote_word`) ולהשוות חזותית את ה-PDF שנוצר מול PDF ייחוס מאותו DOCX.
- הקבצים נשמרים בשרת או באחסון אובייקטים.

יתרונות:

- עימוד זהה או כמעט זהה ל-Word, כי Word עצמו הוא מנוע הפריסה.
- אין חלונות הרשאה למשתמש, כי המנוע רץ בשרת מרוחק ולא על ה-Mac האישי.

חיסרון:

- תפעול כבד ויקר יותר.
- דורש שרת ייעודי או שירות מסחרי conversion.

## חוזה שירות `remote_word`

הקוד תומך בחיבור לשירות Word מרוחק דרך HTTP:

- `POST REMOTE_WORD_URL`
- `multipart/form-data`
- ברירת מחדל של שדה הקובץ: `file`
- ניתן להגדיר כותרת API ופורמט שדות דרך משתני סביבה
- התשובה יכולה להיות:
  - קובץ PDF ישיר
  - JSON עם `download_url`
  - JSON עם `pdf_base64`

כך אפשר לחבר את המערכת ל:

- שרת Windows פנימי עם Word
- שירות conversion מסחרי
- microservice ארגוני שממיר `DOCX -> PDF` עם Word ומחזיר תוצאה

## היעד הסופי

היעד הוא לא רק לתקן את בעיית ההרשאות, אלא לבנות אפליקציה לטלפון שאפשר לעבוד ממנה באופן מלא, כולל יצירת מסמכים וייצוא PDF, בלי שה-Mac יהיה חלק מהשרשרת.
